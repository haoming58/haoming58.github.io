# 1. 双指针

这个在哈希表和字符串，已经被使用，因此，这里是为了再次地巩固。

## 1.1 移除元素

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并原地修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

示例 1: 给定 nums = [3,2,2,3], val = 3, 函数应该返回新的长度 2, 并且 nums 中的前两个元素均为 2。 你不需要考虑数组中超出新长度后面的元素。

示例 2: 给定 nums = [0,1,2,2,3,0,4,2], val = 2, 函数应该返回新的长度 5, 并且 nums 中的前五个元素为 0, 1, 3, 0, 4。

你不需要考虑数组中超出新长度后面的元素。

### 1.1.1 思路

由于这里的标题直接是双指针，因为就直接开始使用这个指针进行代码实践

快指针：寻找新数组的元素 ，新数组就是不含有目标元素的数组, 一个用于遍历

慢指针：指向更新 新数组下标的位置，一个用于新数组的下标

```python

class Solution():
    def remove(self, nums: list[int], val: int) -> int:

        fast = 0  # 按照你的习惯命名，但它实际是负责写入的指针
        
        for i in range(len(nums)):
            if nums[i] != val:

                nums[fast] = nums[i]

                fast += 1  # 成功写入一个元素后，指针往前推一格
                
        return fast  # fast 最后的值，恰好就是有效元素的个数

```

不同的写法：

```python

class Solution:
    def removeElement(self, nums: List[int], val: int) -> int:
        # 快慢指针
        fast = 0  # 快指针
        slow = 0  # 慢指针
        size = len(nums)
        while fast < size:  # 不加等于是因为，a = size 时，nums[a] 会越界
            # slow 用来收集不等于 val 的值，如果 fast 对应值不等于 val，则把它与 slow 替换
            if nums[fast] != val:
                nums[slow] = nums[fast]
                slow += 1
            fast += 1
        return slow

```

```python
# 相向双指针法
# 时间复杂度 O(n)
# 空间复杂度 O(1)
class Solution:
    def removeElement(self, nums: List[int], val: int) -> int:
        n = len(nums)
        left, right  = 0, n - 1
        while left <= right:
            while left <= right and nums[left] != val:
                left += 1
            while left <= right and nums[right] == val:
                right -= 1
            if left < right:
                nums[left] = nums[right]
                left += 1
                right -= 1
        return left

```

下面是这个：

首尾呼应，用右边有用的数字，去填补左边需要删除的空位。

1. 左右包抄（初始化）
动作： 定义两个指针，left 站在数组的最前面（索引 0），right 站在数组的最后面（索引 n-1）。

目的： 两人准备相向而行，从两头向中间巡视整个数组。

2. 左指针找“坑”（第一个 while 循环）
动作： 左指针 left 一路向右走。只要它脚下的数字不是目标值 val，说明这个数字是安全的，不用管它。

触发： 一旦 left 踩到了 val，它就立刻停下。

意义： 此时，左边出现了一个需要被填补的**“坑”**（废弃位置）。

3. 右指针找“替补”（第二个 while 循环）
动作： 右指针 right 一路向左（前）退。只要它脚下的数字是目标值 val，说明这本来就是要扔掉的数字，不用管它。

触发： 一旦 right 踩到了一个不是 val 的正常数字，它也立刻停下。

意义： 此时，右边找到了一个可以用来填坑的**“替补”**（有效数据）。

4. 填坑换位（if left < right: 判断与覆盖）
当左边有“坑”，右边有“替补”时，执行终极连招：

第一步（填坑）： 执行 nums[left] = nums[right]，直接用右边的“替补”把左边的“坑”覆盖掉。

第二步（指针推进）： 填完坑后，双方都完成了一次任务，需要各自走向下一个目标：

left += 1（向右走）： 左指针的坑填满了，索引变大，在尺子上向右走，去检查后面下一个未知的元素。

right -= 1（向左走）： 右指针的替补用过了（已经复制到了前面），失去利用价值。索引变小，在尺子上向左走（往前面走），去寻找前一个还没被动过的“新替补”。

## 1.2 反转字符串II

给定一个字符串 s 和一个整数 k，从字符串开头算起, 每计数至 2k 个字符，就反转这 2k 个字符中的前 k 个字符。

如果剩余字符少于 k 个，则将剩余字符全部反转。

如果剩余字符小于 2k 但大于或等于 k 个，则反转前 k 个字符，其余字符保持原样。

示例:

输入: s = "abcdefg", k = 2
输出: "bacdfeg"
